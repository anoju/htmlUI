현재 제공 받은 피그마 디자인을 보면 다수 텍스트 박스의 높이값 이슈가 있습니다다

예를 들어 3줄 텍스트 박스 영역인데
폰트사이즈 16px, 줄간격 150%로 설정되어있다고 가정하자
그렇게 퍼블리싱 했을때 1줄당 크기가 16px * 1.5 = 24px 이기때문에 전체 텍스트 박스의 크기는 72px이 나와야한다
하지만 피그마 디자인상에는 그보다 훨씬 작은 60px로 되어있습니다.

이때 피그마에서 텍스트 박스 주변 간격을 그대로 css에 적용하면 디자인과 전혀 다른 간격을 갖은 결과물이 나옵니다.
이를 확인하고자 피그마의 Dev Mode를 확인하니 'text-edge' css가 적용되어있었습니다.

피그마에서 Dev Mode 에서 제공하는 'text-edge' css는 해당 css는 최신 css로 현재 크롬 최신버전에서도 지원하지 않습니다.

'text-box-edge' 라는 다른 css로 대처하였으나 여전히 브라우저저 호환성이 떨어집니다.

결론, 텍스트박스의 height가 line-height 크기에 맞게 떨어져야 퍼블리싱할때 이슈가 없습니다.
