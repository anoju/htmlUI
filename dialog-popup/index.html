HTML5 Native Dialog API 마스터

커스텀으로 짜여진 라이브러리 함수가 아닙니다! 순수 HTML5 <dialog> 요소가 자체적으로 제공하는 네이티브 자바스크립트 메서드 3가지의 기능과 기본 포커스 동작 원리를 완벽하게 분석하고 실습하는 환경입니다.

1. showModal() 메서드

  • 백드롭 생성: 배경을 차단(::backdrop)하며 외부 마우스/키보드 조작을 완벽히 통제합니다.
  • 포커스 트랩 (Focus Trap): 열린 직후부터 Tab 키를 눌러도 포커스가 모달 내부에서만 순환하도록 갇힙니다.
  • 오토 포커스: 다이얼로그 내부에 autofocus 속성이 있거나 tabindex인 요소로 자동 포커싱됩니다.

2. show() 메서드

  • Non-modal 형태: 백드롭 오버레이가 생기지 않고 단순히 화면에 요소가 드러납니다.
  • 백그라운드 조작 허용: 팝업이 떠 있는 중에도 뒤쪽의 버튼을 조작하거나 스크롤 할 수 있습니다. 포커스 트랩이 해제됩니다.

3. close([반환값]) 및 기타 API

  • 포커스 복귀: 모달이 닫히면, 열기 직전 활성화되었던 요소(버튼)로 브라우저가 스스로 포커스를 반환해줍니다.
  • 결과값 반환: close('ok') 처럼 파라미터를 넘기면, 이후 자바스크립트에서 dialog.returnValue 로 값을 편하게 읽을 수 있습니다.
  • 네이티브 이벤트: ESC 키를 누르면 브라우저 하드웨어 로직에 의해 cancel 이벤트가 발생하고 모달을 알아서 닫습니다. 추가 메서드 없이 이 3가지만으로 제어됩니다.

4. 부가적인 레이아웃 형태

⚠️ 10년 치 구형 브라우저 실무 지원 시 주의사항

기본적인 <dialog> 기반의 모달은 2022년 상반기(iOS Safari 15.4 이상) 최신 브라우저부터 완벽히 지원됩니다.
만약 2014년~2021년 사이의 구형 아이폰(iOS 14 이하)이나 IE 환경까지 지원해야 한다면 showModal() 호출 시 에러가 발생합니다.

💡 실무 권장 해결책 (Polyfill 사용법):
아래 코드를 HTML 파일과 자바스크립트에 추가하기만 하면, 구형 기기에서도 최신 showModal() 문법이 완벽하게 돌아가게끔 브라우저가 패치됩니다. 
<!-- 1. Polyfill CDN 스크립트 및 CSS 추가 (head 영역에 삽입) -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/dialog-polyfill/0.5.6/dialog-polyfill.min.css">
<script src="https://cdnjs.cloudflare.com/ajax/libs/dialog-polyfill/0.5.6/dialog-polyfill.min.js"></script>

<!-- 2. 자바스크립트에서 다이얼로그 요소 구문 적용 (사용 전 1회 호출) -->
<script>
  const myDialog = document.getElementById('dialogBase');
  dialogPolyfill.registerDialog(myDialog);
</script>
